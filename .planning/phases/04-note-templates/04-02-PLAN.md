  <action>
Create a NoteCreator class that orchestrates the full note creation flow:

**Constructor:** Takes SupernoteAPIClient, NoteTemplateGenerator, FolderMappingStore.

**Method: `createNoteForMeeting(meeting: MeetingData): Promise<void>`**
1. Look up folder mapping via `mappingStore.getMappingByMeetingId(meeting.id)` — if no mapping exists, log warning and skip (folder not yet created)
2. Check if note already created: if mapping has `noteFileName` set, skip (prevent duplicates)
3. Generate markdown content via `templateGenerator.generateMarkdown(meeting)`
4. Generate filename via `templateGenerator.generateFileName(meeting)`
5. Upload via `supernoteClient.uploadTextFile(content, fileName, mapping.folderId)`
6. Update the folder mapping with `noteFileName` field to track that note was created

**Method: `processNewMeetings(meetings: MeetingData[]): Promise<{ created: number; skipped: number }>`**
- Same batch pattern as FolderOrganizer.processNewMeetings()
- 500ms delay between uploads (rate limiting)
- Per-meeting try/catch for error isolation
- Log results

**Extend FolderMapping interface** in folder-mapping-store.ts:
- Add optional `noteFileName?: string` field to FolderMapping interface
- Add `updateMapping(meetingId: string, updates: Partial<FolderMapping>): void` method to FolderMappingStore that finds and updates an existing mapping, then saves

**Tests:**
- createNoteForMeeting: happy path (generates + uploads + updates mapping)
- createNoteForMeeting: skips when no folder mapping exists
- createNoteForMeeting: skips when noteFileName already set (duplicate prevention)
- createNoteForMeeting: handles upload failure gracefully
- processNewMeetings: processes batch with rate limiting
- FolderMappingStore.updateMapping: updates existing mapping
