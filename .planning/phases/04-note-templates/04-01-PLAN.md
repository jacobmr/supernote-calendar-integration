  <files>src/services/supernote-api.ts, tests/supernote-api.test.ts</files>
  <action>
Add an `uploadFile(content: Buffer, fileName: string, directoryId: number)` method to SupernoteAPIClient implementing the 3-step Supernote Cloud upload flow:

1. **Apply**: POST to `/api/file/upload/apply` with `{ directoryId, fileName, md5: md5HashOfContent, size: contentByteLength }`. Response contains `{ url, s3Authorization, xamzDate, innerName }` (S3 pre-signed upload details).

2. **PUT to S3**: PUT the raw content buffer to the `url` from step 1, with headers:
   - `Authorization`: value from `s3Authorization`
   - `x-amz-date`: value from `xamzDate`
   - `x-amz-content-sha256`: `"UNSIGNED-PAYLOAD"`
   - `Content-Type`: `application/octet-stream`

3. **Finish**: POST to `/api/file/upload/finish` with `{ directoryId, fileName, fileSize: contentByteLength, innerName: innerNameFromApply, md5: md5HashOfContent }`.

Use Node.js built-in `crypto.createHash('md5')` for MD5 calculation. Use the existing `postJson()` helper for steps 1 and 3. For step 2, use native `fetch()` directly (it's a PUT to an external S3 URL, not the Supernote API).

Add a convenience method `uploadTextFile(content: string, fileName: string, directoryId: number)` that converts string to Buffer and calls uploadFile.

Add unit tests mocking the 3-step flow: test apply call, S3 PUT, finish call, and error handling (apply failure, S3 failure, finish failure). Do NOT test against real API.

Create a NoteTemplateGenerator class with a `generateMarkdown(meeting: MeetingData): string` method that produces structured markdown meeting notes.

**Template structure:**
```markdown
# [Meeting Title]

**Date:** [formatted date, e.g. "Tuesday, March 18, 2026"]
**Time:** [formatted time range, e.g. "10:00 AM - 11:00 AM"]
**Location:** [location if available, omit section if not]

## Attendees

- [DisplayName] (email@example.com) - [responseStatus]
- [DisplayName] (email@example.com) - [responseStatus]

## Agenda

[Description from calendar event if available, otherwise: "No agenda provided"]

---

## Notes



## Action Items

- [ ]
- [ ]
- [ ]

## Decisions



---
*Generated by Supernote Calendar Integration*
```

**Formatting rules:**
- Use `Intl.DateTimeFormat` for date/time formatting (no extra dependencies)
- Attendee responseStatus: map "accepted" → "✓", "declined" → "✗", "tentative" → "?", "needsAction" → "pending"
- If attendees array is empty, show "No attendees listed"
- If description contains HTML tags, strip them (simple regex: `/<[^>]*>/g` replace with empty)
- Trim description to first 2000 chars if very long

Also add a `generateFileName(meeting: MeetingData): string` method:
- Recurring meetings: `YYYY-MM-DD [Sanitized Title].md` (date from meeting startTime)
- Ad-hoc meetings: `YYYY-MM-DD HH-MM [Sanitized Title].md` (include time to avoid collisions in shared folder)
- Sanitize: replace `<>:"|?*\/` with `_`, limit to 100 chars total
- Use the same sanitization approach as FolderOrganizer.sanitizeFolderName()

Write comprehensive tests:
- generateMarkdown with full MeetingData (all fields populated)
- generateMarkdown with minimal MeetingData (no description, no attendees, no location)
- generateMarkdown with HTML description (tags stripped)
- generateFileName for recurring meeting
- generateFileName for ad-hoc meeting
- generateFileName with special characters in title
