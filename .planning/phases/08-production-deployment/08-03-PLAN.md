    ```
    ssh root@37.27.198.218 'bash -s' << 'F2B_SCRIPT'
    set -e
    apt-get update -qq && apt-get install -y -qq fail2ban

    # Create local config (overrides defaults without modifying originals)
    cat > /etc/fail2ban/jail.local << 'EOF'
    [DEFAULT]
    # Ban for 10 minutes (not forever)
    bantime = 600
    # Allow 10 failed attempts (generous — prevents accidental lockout)
    maxretry = 10
    # Within a 10-minute window
    findtime = 600
    # Whitelist ZeroTier subnet so VPN access is never banned

    port = ssh
    filter = sshd
    logpath = /var/log/auth.log
    maxretry = 10
    EOF

    systemctl enable fail2ban
    systemctl restart fail2ban
    fail2ban-client status sshd
    F2B_SCRIPT
    ```

    **Why these settings are safe:**
    - maxretry=10: Very generous — won't trip on normal SSH usage
    - bantime=600: 10-minute bans, not permanent — recovers automatically
    - ignoreip includes all RFC1918 ranges: ZeroTier IPs (typically 10.x.x.x or 172.x.x.x) are never banned
    - Even if public IP gets banned, ZeroTier SSH works (different interface, whitelisted subnet)
