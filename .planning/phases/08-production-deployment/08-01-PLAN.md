<task type="checkpoint:human-action" gate="blocking">
  <action>Gather real production credentials</action>
  <instructions>
    Claude needs the following real values to update SOPS secrets. Please provide them:

    **Google Calendar OAuth2** (from https://console.cloud.google.com → APIs & Services → Credentials):
    1. `GOOGLE_CLIENT_ID` — your OAuth2 client ID
    2. `GOOGLE_CLIENT_SECRET` — your OAuth2 client secret
    3. Verify the authorized redirect URI includes `http://localhost:3000/oauth/callback` (for local token bootstrap — production doesn't need its own redirect URI because tokens are bootstrapped locally and copied to server)

    **Supernote** (your Supernote account):
    4. `SUPERNOTE_EMAIL` — your Supernote login email
    5. `SUPERNOTE_PASSWORD` — your Supernote password

    **Dashboard**:
    6. `DASHBOARD_PASSWORD` — choose a strong password for the web dashboard at supernote.salundo.com

    **Timezone**:
    7. `TZ` — your timezone (default: America/New_York)
  </instructions>
  <resume-signal>Provide all credential values (they'll be encrypted with SOPS immediately, never stored in plaintext)</resume-signal>
</task>

<task type="auto">
  <name>Task 2: Update SOPS-encrypted secrets with real values and deploy</name>
  <files>secrets.production.env.enc</files>
  <action>
    1. Decrypt the SOPS file: `sops secrets.production.env.enc`
    2. Replace all placeholder values with the real credentials provided by the user
    3. Add `TOKEN_STORE_PATH=/app/data/tokens.json` so tokens persist in Docker volume across rebuilds
    4. Set `GOOGLE_REDIRECT_URI=http://localhost:3000/oauth/callback` (OAuth tokens are bootstrapped locally, not in production)
    5. Save and let SOPS re-encrypt
    6. Run `scripts/deploy.sh` to deploy with real credentials
    7. Verify health check passes at https://supernote.salundo.com/health

    **Important:** The deploy script pipes decrypted secrets via SSH — no plaintext ever touches disk. After deploy, the .env on the server has the real values and the container can start. But Google Calendar won't work yet until OAuth tokens are bootstrapped (Plan 08-02).
  </action>
  <verify>
    - `curl -sf https://supernote.salundo.com/health` returns `{"status":"ok"}`
    - `ssh root@37.27.198.218 'grep -c TOKEN_STORE_PATH /opt/supernote/.env'` returns 1
    - Dashboard login works with new password at https://supernote.salundo.com/login
  </verify>
  <done>App deployed with real credentials, health check passing, dashboard accessible with new password. OAuth token bootstrap pending (Plan 08-02).</done>
</task>
