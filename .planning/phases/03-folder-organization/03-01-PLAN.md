  <name>Task 2: Create FolderMappingStore for meeting-to-folder persistence</name>
  <files>src/services/folder-mapping-store.ts, tests/folder-mapping-store.test.ts</files>
  <action>
1. Create `src/services/folder-mapping-store.ts`:
   - Interface `FolderMapping`: `{ meetingId: string, meetingTitle: string, folderId: number, folderPath: string, isRecurring: boolean, createdAt: number }`
   - Interface `RecurringFolderMapping`: `{ recurringEventId: string, meetingTitle: string, folderId: number, folderPath: string, createdAt: number }` — for deduplicating recurring meeting instances that share one folder
   - Interface `FolderMappingState`: `{ mappings: FolderMapping[], recurringFolders: RecurringFolderMapping[] }`
   - Class `FolderMappingStore` following StateManager pattern:
     - Constructor takes filepath (default `data/folder-mappings.json`)
     - `loadState(): FolderMappingState` — read from disk, return empty arrays if missing
     - `saveState(state: FolderMappingState): void` — write to disk, create dir if needed
     - `getMappingByMeetingId(id: string): FolderMapping | undefined`
     - `getRecurringFolder(recurringEventId: string): RecurringFolderMapping | undefined`
     - `addMapping(mapping: FolderMapping): void` — load, add, save
     - `addRecurringFolder(mapping: RecurringFolderMapping): void` — load, add, save
     - `hasFolderForMeeting(meetingId: string): boolean`
     - `hasFolderForRecurringEvent(recurringEventId: string): boolean`

   Follow the exact same file I/O pattern as StateManager (fs.existsSync, fs.readFileSync, fs.writeFileSync, fs.mkdirSync with recursive).

2. Create `tests/folder-mapping-store.test.ts`:
   - Test loadState returns empty state when file missing
   - Test addMapping persists and is retrievable
   - Test addRecurringFolder persists and is retrievable
   - Test hasFolderForMeeting returns true/false correctly
   - Test hasFolderForRecurringEvent returns true/false correctly
