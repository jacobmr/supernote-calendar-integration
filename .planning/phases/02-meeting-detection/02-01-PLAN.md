   - Query current meetings via MeetingDetectorService
   - Compare previous vs current to detect changes:

     - Changed meetings: same ID but different time/attendees
     - Cancelled meetings: in previous, not in current
   - Log all changes with details (new count, changed count, cancelled count)
   - Save new state after detection completes
   - Format logs clearly for monitoring (timestamps, counts, affected meetings)
