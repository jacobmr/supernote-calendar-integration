  <action>
  Create Express server listening on port 3000 with:
  1. Password protection middleware: Check for `password` cookie; if missing, redirect to `/login`
  2. `/login` POST endpoint: Accept password in form, validate against environment variable `DASHBOARD_PASSWORD` (default "admin" for local dev), set httpOnly cookie on success
  3. `/setup` GET route: Serve setup form HTML (will create in Task 2)
  4. `/dashboard` GET route: Serve dashboard HTML (will create in next plan)
  5. Root `/` redirects to `/setup` on first visit, `/dashboard` after setup
  6. Graceful shutdown when parent process exits (for Docker integration)

  Use simple sessions via httpOnly cookies (no session store needed for single-user app). Validate password logic: if DASHBOARD_PASSWORD not set in env, use default "admin" (local dev only).

  **What to avoid:** Do NOT use passport.js or complex auth frameworks—too heavyweight for simple password. Keep it minimal.
  </action>
