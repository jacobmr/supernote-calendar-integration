# Phase 2.2 Plan 02: Dashboard UI & Docker Integration Summary

**Built meeting summary dashboard with color-coded metrics, API status endpoint, and Docker integration running both Express server and scheduler in a single container.**

## Accomplishments

- Dashboard HTML template displaying meeting state summary with three color-coded metric cards (new/changed/cancelled)
- `/api/status` GET endpoint reading scheduler state from `data/scheduler-status.json`
- `/logout` GET endpoint clearing authentication cookie and redirecting to login
- `/api/trigger` POST placeholder for future manual scheduler trigger
- Scheduler updated to write `data/scheduler-status.json` after each run with change summary
- Dockerfile updated to run both Express server and scheduler via `start.sh`
- Docker image builds and runs successfully with both services in a single container
- ESM compatibility fix for `supernote-cloud-api` module (dynamic import)

## Files Created/Modified

- `src/index-server.ts` - Replaced placeholder dashboard with full meeting summary UI, added `/api/status`, `/logout`, `/api/trigger` endpoints, fixed ESM import
- `src/index-scheduler.ts` - Added `fs`/`path` imports and scheduler-status.json write after each run
- `Dockerfile` - Updated to install all deps for build then prune, copy config dir, use `start.sh` entrypoint
- `start.sh` - New startup script running both Express server and scheduler with signal handling
- `package.json` - Added `start` script

## Decisions Made

- Created `data/scheduler-status.json` as a separate file from `meeting-state.json` to store dashboard-facing summary (lastRun, change counts, status) without modifying the state file format
- Used dynamic import for `supernote-cloud-api` to avoid ESM/CommonJS incompatibility in compiled output
- Used shell script (`start.sh`) for multi-process management instead of `concurrently` package to avoid adding dependencies
- Dashboard uses inline HTML/CSS/JS (no separate files) consistent with existing login/setup pages
- Scheduler runs in background, Express server runs in foreground -- both managed by `start.sh` with trap for graceful shutdown
- Dashboard shows relative time ("3 minutes ago") for last run timestamp

## Issues Encountered

1. **Docker build: tsc not found** - Original Dockerfile installed `--only=production` which excludes TypeScript. Fixed by installing all deps, building, then pruning with `npm prune --production`.
2. **Docker build: missing config directory** - `config/google-api.ts` was not copied to Docker image. Added `COPY config ./config` to Dockerfile.
3. **Docker runtime: compiled JS path** - `tsconfig.json` has `rootDir: "./"` so compiled files land at `dist/src/` not `dist/`. Updated `start.sh` to use `dist/src/index-server.js` and `dist/src/index-scheduler.js`.
4. **ESM compatibility** - `supernote-cloud-api` is ESM-only; top-level `require()` in compiled CommonJS fails. Converted to dynamic `import()` in the setup route handler.
